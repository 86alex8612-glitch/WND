# Описание файлов проекта «НейроКонсультант по ВНД»

Документ описывает назначение каждого файла приложения и переиспользуемые блоки кода (функции, классы, модули).

**Примечание:** термин «функция» далее используется в широком смысле — как обозначение переиспользуемого блока кода, включая функции, методы классов, классы и модули.

---

## Корневая папка проекта

### `start_server.bat`
Скрипт запуска backend-сервера (uvicorn) на Windows.

### `config.cfg`
Файл конфигурации рабочей папки данных (`data_root`). Используется модулем `path_config.py`.

### `env_template.txt`
Шаблон переменных окружения (API-ключ OpenAI, путь к данным и др.) для создания файла `.env`.

### `.env` (не для GitHub)
Локальные секреты и настройки окружения. Создаётся пользователем на основе `env_template.txt`.

### `.gitignore`
Список файлов и папок, исключаемых из системы контроля версий.

### `backend/requirements.txt`
Список Python-зависимостей backend (FastAPI, ChromaDB, LangChain, OpenAI и др.).

### `ИНСТРУКЦИЯ_ПОЛЬЗОВАТЕЛЯ.md`
Полная пользовательская инструкция по работе с приложением. Отдаётся через API `/api/user-instruction` и показывается на главной странице.

### `DEPLOY_BEGET.md`
Инструкция по развёртыванию приложения на хостинге Beget.

### `Промпты_Анализ_ВНД.txt`
Текстовый архив/справочник промптов для анализа ВНД (исходные формулировки).

### `Ответы_три_базы_источники_алгоритмы.txt`
Справочный документ: описание трёх векторных баз (ГОСТ, ФЗ, ВНД), источников данных и алгоритмов поиска.

### `prompts/промпт инструкция.txt`
Текстовый файл с промптом для генерации инструкций.

### `prompts/доработка создание документы.txt`
Текстовый файл с промптами для доработки и создания документов.

### Рабочие папки данных (создаются автоматически)
| Папка | Назначение |
|-------|------------|
| `IN/` | Исходные ВНД для анализа, поиска и переработки |
| `IN/create/` | Файлы, загруженные в «Помощник в создании ВНД» |
| `IN/search/` | Файлы, загруженные в «Поиск в ВНД» |
| `FZYur/` | ГОСТ и отраслевые стандарты |
| `FZ/` | Федеральные законы |
| `OUT/` | Сохранённые отчёты правового анализа |
| `new-doc/` | Резервная папка проектов документов на сервере |
| `logs/` | Логи работы приложения |
| `backend/chroma_db/` | Локальные векторные индексы ChromaDB |
| `backend/wnd.db` | База SQLite (диалоги) |

### `backend/main.py`
**Назначение:** точка входа FastAPI-приложения. Маршрутизация HTTP-запросов, отдача HTML-страниц, REST API для всех разделов.

- `Stage1Request`, `AnalyzeVndRequest` — анализ ВНД

- `ReportSaveRequest` — сохранение отчёта анализа
- `WorkFolderRequest`, `BrowseFolderRequest` — настройки рабочей папки
- `SearchVndRequest`, `SearchLoadDocumentRequest`, `SearchQAMessageRequest`, `SearchQASaveRequest` — поиск в ВНД

- `root`, `analiz_page`, `search_page`, `create_page` — маршруты HTML-страниц
- `user_instruction` — API: текст `ИНСТРУКЦИЯ_ПОЛЬЗОВАТЕЛЯ.md`
- `ui_version` — диагностика версии frontend
- `init`, `get_unfinished_dialogs`, `get_dialog`, `create_dialog`, `send_message`, `continue_dialog`, `remove_dialog` — API диалогов (legacy)
- `_save_uploaded_file`, `_file_download_response`, `upload_vnd`, `_handle_vnd_upload` — загрузка ВНД
- `_tkinter_browse_vnd_file`, `browse_vnd_file` — выбор файла через диалог ОС

- `vnd_stage1`, `vnd_analyze` — этапы анализа ВНД

- `create_rework_stage1`, `create_rework_analyze`, `create_rework_generate`, `create_rework_run` — переработка документа

## Frontend — JavaScript-модули

### `frontend/errors.js`
**Назначение:** общая обработка ошибок API и сетевых сбоев.

**Функции:**
- `humanizeErrorMessage` — преобразование текста ошибки
- `readApiError` — чтение ошибки из HTTP-ответа
- `formatCaughtError` — форматирование исключения для UI

**Экспорт в `window`:** `formatCaughtError`, `readApiError`, `humanizeErrorMessage`

---

### `frontend/downloads.js`
**Назначение:** скачивание файлов через POST-запросы к API.

**Функции:**
- `parseFilenameFromDisposition` — имя файла из заголовка Content-Disposition
- `triggerBrowserDownload` — сохранение blob в браузере
- `downloadFromPost` — универсальное скачивание через API

---

### `frontend/config-ui.js`
**Назначение:** модальное окно настроек — рабочая папка и подпапки.

**Функции:**
- `fetchAppConfig` — загрузка конфигурации с сервера
- `renderSettingsForm` — отображение формы настроек
- `saveWorkFolderSettings`, `resetWorkFolderSettings`, `browseWorkFolder` — сохранение, сброс, выбор папки
- `openSettingsModal`, `closeSettingsModal`, `initSettingsGear` — управление модальным окном
- `uploadFilesToFolder`, `saveDocumentToWorkFolder` — вспомогательные загрузки

---

### `frontend/user-instruction.js`
**Назначение:** модальное окно «Инструкция пользователя» на главной странице (из `ИНСТРУКЦИЯ_ПОЛЬЗОВАТЕЛЯ.md`).

**Функции:**
- `renderInstructionMarkdown` — преобразование Markdown в HTML
- `loadUserInstructionContent` — загрузка текста с API
- `openUserInstructionModal`, `closeUserInstructionModal` — управление окном
- `bindUserInstructionHandlers` — привязка событий

---

### `frontend/app.js`
**Назначение:** логика главной страницы — навигация, карточка «Статистика», управление базами.

**Функции:**
- `openAnalizPage`, `openCreatePage`, `openSearchPage` — переход к разделам
- `updateStatsInCard`, `renderStatItem` — отображение статистики баз
- `openBaseDocumentsModal`, `resetBase`, `recreateBase`, `recreateBases` — просмотр и обслуживание баз
- `uploadStatsFiles`, `initStatsCardHandlers` — загрузка ГОСТ/ФЗ с компьютера

---

### `frontend/analiz.js`
**Назначение:** клиентская логика раздела «Анализ ВНД».

**Основные функции:**
- `uploadDocument`, `browseVndDocument`, `pickAnalizFile` — загрузка ВНД
- `checkFederalReferences`, `downloadFederalDocs`, `showFederalRefsPanel` — федеральные ссылки
- `proceedWithStage1`, `submitStage1`, `populateStage1Form`, `collectStage1Answers` — этап 1
- `proceedWithStage1` → правовой анализ — этап 2
- `saveReportToWorkFolder`, `downloadReport` — сохранение отчёта
- `loadUnfinishedDialogs`, `deleteUnfinishedDialog` — незавершённые диалоги (legacy)

---

### `frontend/create.js`
**Назначение:** клиентская логика «Помощник в создании ВНД».

**Основные функции:**
- `showReworkMode`, `showNewMode`, `showReworkStep`, `showNewStep` — навигация по шагам
- `uploadCreateFile`, `autoLoadReworkAnalysis`, `goToReworkForm` — переработка: загрузка и подбор отчёта
- `collectReworkFormData`, `runReworkFromForm`, `runReworkAnalyzeAndGenerate` — переработка: анкета и генерация
- `collectNewFormData`, `runNewAnalyze`, `runNewContinue`, `runNewGenerate` — новый документ: анкета и генерация
- `renderFollowupQuestions`, `validateFollowupClient` — уточняющие вопросы
- `downloadCreateDocument` — скачивание DOCX
- `openCreateInstructionModal`, `closeCreateInstructionModal` — инструкция по карточке
- `openCreateQaModal`, `sendCreateQaMessage`, `downloadCreateQaDialog` — диалог «Задать вопросы»

**Модульный блок:**
- `CREATE_INSTRUCTION_HTML` — HTML-текст инструкции помощника

### `frontend/search.js`
**Назначение:** клиентская логика «Поиск в ВНД».
